# AgentChange

AgentChange records activity observed through supported Codex lifecycle hooks. This skill is an instruction layer, not the evidence source.

1. Understand the requested task before editing. Inspect the initial Git status and keep pre-existing changes separate from files created or modified during this turn.
2. Run relevant validation where practical. For Python validation, use `agentchange exec --auto <command> [arguments...]` so AgentChange selects the project Python and independently observes the subprocess exit code.
   - `agentchange exec --auto pytest -q`
   - `agentchange exec --auto python -m pytest -q tests/test_example.py`
   - `agentchange exec --auto ruff check .`
   Do not require the wrapper for ordinary read-only commands such as `ls`, `cat`, `sed`, or `git status`.
3. Never claim tests passed unless a successful result was observed.
4. State failures and unresolved issues honestly.
5. Before the final response, inspect the Git diff only for repository-relative files created or modified during this turn. Do not include files that were only pre-existing changes. Summarize the requested task, important changes, validation performed, known failures, assumptions, and suggested reviewers.
6. End the final response with this exact plain-text block, using valid JSON and one concise entry per same-turn changed file where practical. Describe only meaningful changes visible in the inspected diff; do not guess intent or include unsupported paths.

```text
AGENTCHANGE_CHANGE_SUMMARY_JSON
{
  "file_summaries": [
    {
      "path": "repository/relative/path",
      "summary": "One concise sentence describing the meaningful code change."
    }
  ]
}
END_AGENTCHANGE_CHANGE_SUMMARY_JSON
```

These descriptions are generated by Codex from the observed Git diff. They remain model-derived statements and are never observed evidence.
7. Do not manually send a receipt; the Stop hook creates the per-turn local receipt and handles configured Slack delivery.
8. Do not modify AgentChange runtime evidence files under `PLUGIN_DATA`.

Validation results are considered independently observed only when executed through `agentchange exec --auto`, the backward-compatible `agentchange-run`, or another supported tool that provides an explicit reliable result.

AgentChange evidence is not complete, tamper-proof, a full audit trail, or a secure enforcement boundary.
