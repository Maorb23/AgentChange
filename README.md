# AgentChange for Codex — Phase 1

AgentChange records activity observed through supported Codex lifecycle hooks. Phase 1 passively preserves sanitized hook payloads as per-session JSONL and provides downstream Pydantic normalization.

It is not complete, tamper-proof, a full audit trail, or a secure enforcement boundary. It does not block tools, rewrite inputs, decide permissions, inspect Git, score risk, generate receipts, or send Slack messages.

## Evidence labels

- **Observed:** directly present in a supported hook payload, such as a command, tool response, prompt, or Stop event.
- **Inferred:** conservatively derived from observed text, such as a patch path or an unambiguous textual exit code.
- **Reported:** a statement generated by Codex, including `Stop.last_assistant_message`.
- **Unknown:** not reliably determined. Missing exit codes and permission decisions remain unknown.

## Captured events and storage

The plugin registers `SessionStart`, `UserPromptSubmit`, `PreToolUse`, `PermissionRequest`, `PostToolUse`, and `Stop`. Tool hooks use catch-all matching and can see supported Bash, `apply_patch`, MCP, and other local function paths.

Runtime data is written only under:

```text
${PLUGIN_DATA}/sessions/<sanitized-prefix>-<session-id-sha256>/
├── events.jsonl
├── metadata.json
└── normalization_errors.jsonl
```

`events.jsonl` contains sanitized raw envelopes. The hook path uses only the Python standard library. Pydantic normalization happens after raw evidence is preserved.

Strings are limited to 8,192 characters, collections to 100 items, and nesting to eight levels. Common credential-like keys, bearer values, environment assignments, and Slack webhook URLs are redacted. Detection is intentionally basic and cannot guarantee removal of every secret.

## Local installation and trust

The installer copies the plugin to the documented personal source at `~/plugins/agentchange` and registers it in `~/.agents/plugins/marketplace.json` without removing other entries.

```powershell
python scripts\install_local.py
codex plugin add agentchange@personal
```

Start a new Codex session after installation. Open `/hooks`, inspect the source and exact commands, then trust the AgentChange hooks. Hooks do not run merely because the plugin was installed or enabled.

To disable or uninstall, open `/plugins`, select AgentChange, and disable or uninstall it. Remove the personal marketplace entry and `~/plugins/agentchange` only if you also want to remove the development source.

## Diagnostics

Fixture mode uses the same capture logic and prints the normalized event:

```powershell
$data = Join-Path $env:TEMP "agentchange-test"
python -m agentchange.hook_entry capture `
  --fixture fixtures\codex_hooks\post_tool_use_bash_failed.json `
  --plugin-data $data
```

Generate the non-risk-scored sample session:

```powershell
python scripts\generate_demo_events.py
```

The output is `examples/sample_session.jsonl`; it is regenerated through production capture and normalization rather than maintained by hand.

## Known observation gaps

AgentChange records activity observed through supported Codex lifecycle hooks. Hosted tools, specialized tool paths, actions outside Codex, and activity while hooks are disabled or failing may not be captured.

`PermissionRequest` shows that approval was requested, not the final human decision. `PostToolUse.tool_response` is tool-specific, so an exit code remains unknown unless a supported structured field or unambiguous textual form is present. Local files can be modified, and hook evidence is not remote attestation.
